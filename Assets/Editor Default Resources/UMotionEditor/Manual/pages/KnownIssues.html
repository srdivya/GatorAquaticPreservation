Known Issues

This page lists all known issues that can't be fixed. These issues are usually related to a specific Unity version.

Issue 01: Shortcuts in Context Menus and Menu Bar not displayed

In Unity 5.3 shortcuts aren't displayed correctly in context menus and in the menu bar of the Clip Editor. This is due to a bug in that Unity version and is fixed in newer Unity versions.

Issue 02: Bones are rendered with a visible offset and often appear above the Unity editor UI.

This issue is related to Unity 5.6.1f1. Please update to Unity 5.6.2f1 or newer.

Issue 03: In the Animation Event window the tooltip of the warning icon is not working correctly.

The warning icon usually shows a detailed warning message in its tooltip. The tooltip is not working when the window is opened and no UI elment has been clicked in the window. After any UI element was clicked, the tooltip is working correctly. This issue is related to Unity 5.3.5f1 but may also be present in other Unity version. In Unity 2017.1.0f1 the tooltip is working as expected.

Issue 04: Humanoid export produces visible differences.

When a humanoid animation clip is exported, the exported clip has notable differences (most likely at the fingers) compared to the UMotion animation clip. This issue is related to a bug in all Unity 5.3 and Unity 5.4 versions. Please update to Unity 5.5.0f2 or newer.

Issue 05: Exporting Humanoid IK curves is not supported.

A warning message mentioning this known issue is displayed every time a humanoid animation is exported. It is just for informational pupose and usually can be safely ignored.

Exporting humanoid hand/foot IK curves is not supported by Unity at the moment. These curves are only necessary when the Mecanim foot/hand IK is used to avoid hand/foot sliding. More information: Import / Export

As soon as this feature is implemented in Unity, it will be available in UMotion.

Issue 06: Humanoid export produces invalid RootT curves when hips or any hips parent bone is scaled.

When a humanoid animation clip is exported the root motion (RootT curve) isn't exported correctly when the bone that is mapped to the humanoid "Hips" or any of its parent bones is scaled (!= 1) as seen in the following screenshot:



This is related to a Unity bug that is present in all current Unity versions. The bug is already reported and will be fixed soon.
